# ARVO2.0 개발 가이드라인

## 📁 문서 관리 규칙

### 문서 위치 원칙:

#### Root (필수 문서만!)
```
✅ README.md           - 프로젝트 개요
✅ QUICK_START.md      - 빠른 시작
✅ VERSION_HISTORY.md  - 버전 히스토리
✅ CHANGES.md          - 상세 변경사항
✅ DOCUMENT_MAP.md     - 문서 네비게이션
```

**규칙**: 
- 최종 완성된 문서만
- 사용자가 바로 읽을 문서만
- 최대 5-7개 유지

---

#### docs/working/ (중간 작업 문서) ⭐
```
📝 DOCUMENTATION_SUMMARY.md    - 문서 정리 과정
📝 FINAL_ORGANIZATION.md       - 파일 구조 정리
📝 CLEANUP_SUMMARY.md          - 정리 요약
📝 *_ANALYSIS.md               - 분석 과정 문서
📝 *_REVIEW.md                 - 검토 과정 문서
```

**규칙**:
- 작업 과정 문서
- 중간 요약/분석
- 완성되지 않은 문서
- 개발자만 참고

---

#### docs/daily/ (일일 요약)
```
📅 TODAY_IMPROVEMENTS_SUMMARY.md
📅 FINAL_SUMMARY_YYYYMMDD.md
```

**규칙**:
- 하루 작업 완료 후 작성
- 날짜별 관리
- 최종 완성본만

---

#### docs/analysis/ (최종 분석)
```
📊 FINAL_PIPELINE_REVIEW.md
📊 *_COMPLETE_ANALYSIS.md
```

**규칙**:
- 완성된 분석만
- 최종 결론 포함
- 재사용 가능

---

#### docs/archive/ (과거 문서)
```
📦 중복 문서
📦 구버전 문서
📦 대체된 문서
```

**규칙**:
- 더 이상 사용 안하는 문서
- 역사적 가치만 있음

---

#### v2.x/ (버전별 문서)
```
📚 00_INDEX.md
📚 01-05_*.md
```

**규칙**:
- 버전별로 분리
- 완성된 문서만
- 버전 릴리스 시 생성

---

## 🔄 작업 프로세스

### 1. 작업 시작
```
docs/working/에 작업 문서 생성
예: WIP_FEATURE_NAME.md
```

### 2. 작업 중
```
docs/working/에서 계속 업데이트
중간 분석, 요약 등 자유롭게
```

### 3. 작업 완료
```
최종 문서 작성:
- 필수면 → Root/
- 일일 요약 → docs/daily/
- 분석 완료 → docs/analysis/
- 버전별 → v2.x/

중간 문서:
- docs/working/에 보관 (참고용)
```

### 4. 버전 릴리스
```
1. v2.x/ 새 디렉토리 생성
2. 완성된 문서 이동
3. VERSION_HISTORY.md 업데이트
4. README.md 업데이트
```

## 📝 문서 작성 규칙

### 파일명:
- **Root**: UPPERCASE_WITH_UNDERSCORES.md
- **working**: WIP_*, TEMP_*, *_DRAFT.md
- **daily**: TODAY_*, FINAL_SUMMARY_*.md
- **analysis**: FINAL_*, *_COMPLETE_*.md
- **v2.x**: 00_*, 01_*, etc.

### 내용:
- 명확한 제목
- 날짜/버전 정보
- 목차 (긴 문서)
- 코드 블록 언어 지정
- 표로 비교
- 이모지로 강조 (✅ ❌ 🎯 등)
